# Contributing

Thanks for your interest in contributing to [mitos-run](https://github.com/mitos-run).
This is the organization wide default; a repository with its own
`CONTRIBUTING.md` overrides it.

## Ways to help

- **Report bugs and request features** through the issue tracker of the relevant
  repository.
- **Improve docs.** Small fixes (typos, clarifications, examples) are very
  welcome and easy to review.
- **Send pull requests.** For anything non trivial, open an issue first so we can
  agree on the approach before you invest time.
- **Help others** in [Discussions](https://github.com/mitos-run/mitos/discussions):
  answer a question, share what you built, or propose an idea.

## Pull request basics

1. Fork and create a branch from `main`.
2. Keep changes focused; one logical change per pull request.
3. Add or update tests and docs alongside the code.
4. Make sure the repository's checks (build, tests, lint) pass locally.
5. Write a clear description: what changed, why, and how you verified it.

## Core runtime

For the core runtime, the detailed developer guide, build instructions, and
coding conventions live in
[mitos-run/mitos/CONTRIBUTING.md](https://github.com/mitos-run/mitos/blob/main/CONTRIBUTING.md).

## Conduct and security

By participating you agree to our [Code of Conduct](CODE_OF_CONDUCT.md). Never
report security issues in public; follow the [Security Policy](SECURITY.md)
instead.
